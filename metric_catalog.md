# 指標カタログ — 不動産買取再販ダッシュボード PoC

## 目的
本ダッシュボードは、**数字の見える化・比較（支店間・個人間・昨年対比）を行い、改善箇所を具体的にすることで、現場の改善活動を促進すること**を目的とする。
各画面で使用する指標を、算式・粒度・閾値・Next Actionテンプレまで含めて一意に定義する。閾値は `config/thresholds.yaml` に外出しされており、本ドキュメントでは「仮置き値」と「根拠」のみ記載する。

## 読み方
| 列 | 内容 |
|---|---|
| 指標名 | ダッシュボード上の表示名 |
| 算式 | 計算式（テーブル・カラムレベル） |
| 粒度 | 集計の最小単位（月 / 個人 / 支店 / 属性 など） |
| 閾値（仮置き） | thresholds.yaml の初期値 |
| 閾値の根拠 | 仮置き値を選んだ理由 |
| ヒアリング状況 | 未 / 実施中 / 確定 |
| Next Action テンプレ | 閾値超過時・条件合致時に画面に表示される文言 |

※「平均」は外れ値の影響を考慮し、**中央値併記**を基本とする。
※すべての指標において**「他支店・他メンバーとの比較」「昨年同月（同期）比較」**の切り口を標準装備する。

## 【1】情報入り口分析
支店・個人ごとの案件獲得経路を分析し、情報源の質と量の変化、および個人の成長に伴うパイプラインの変化を可視化する。

### 1-1. 情報源別 平均案件数
*   **算式**：`COUNT(fact_案件.案件ID) / 営業月数` （GROUP BY 情報源ID）
*   **粒度**：月 / 支店 / 個人 / 入社経過月 / 情報源
*   **閾値（仮置き）**：自支店内で下位情報源は平均の50%未満
*   **根拠**：チャネルごとの安定的な案件流入量を担保するため
*   **ヒアリング状況**：未
*   **Next Action**：「情報源[X]からの月間平均案件数が[閾値]件を下回っています（前年比[Y]%減）。チャネル運用を見直してください」

### 1-2. 情報源別 平均粗利
*   **算式**：`SUM(fact_契約.粗利_確定) / COUNT(DISTINCT fact_案件.案件ID)` （GROUP BY 情報源ID）
*   **粒度**：月 / 支店 / 個人 / 情報源
*   **閾値（仮置き）**：全情報源平均の50%未満
*   **根拠**：情報源ごとの「質（収益性）」を比較するため
*   **ヒアリング状況**：未
*   **Next Action**：「情報源[X]の平均粗利が他平均の[閾値]%以下です。薄利多売の構造になっていないか確認し、投下リソースを見直してください」

### 1-3. 情報源別 成約率
*   **算式**：`COUNT(WHERE 成約フラグ=TRUE) / COUNT(fact_案件)` （GROUP BY 情報源ID）
*   **粒度**：月 / 支店 / 個人 / 情報源
*   **閾値（仮置き）**：17期実績の第1四分位を下回ったら要注意
*   **根拠**：情報源ごとの歩留まりを評価し、無駄な追客を防ぐため
*   **ヒアリング状況**：未
*   **Next Action**：「情報源[X]の成約率が[閾値]%を下回っています。失注理由の傾向と情報源ミックスを確認してください」

### 1-4. 入社経過月数別 情報源ポートフォリオ推移
*   **算式**：`dim_社員.入社からの経過月` と `各情報源の案件割合` のクロス集計
*   **粒度**：個人 × 入社経過月（年次）
*   **閾値（仮置き）**：入社2年目以降で単一（初心者向け）情報源依存が60%超
*   **根拠**：1年目の情報源から、2年目以降の自立した情報源開拓へシフトできているか確認するため
*   **ヒアリング状況**：未
*   **Next Action**：「[個人名]の[情報源]依存度が2年目において[閾値]%を超えています。より難易度・粗利の高い情報源への開拓を指導してください」

## 【2】売却先出口分析
仕入れた物件をどのように販売しているか、支店・個人ごとの出口戦略と健全性を可視化する。

### 2-1. 法人・個人 売却先割合
*   **算式**：`COUNT(fact_契約 WHERE 契約種別='売却') GROUP BY dim_出口.法人個人区分 / 全売却契約数`
*   **粒度**：月 / 支店 / 個人
*   **閾値（仮置き）**：昨年同月または全社平均から±15%以上の乖離で警告
*   **根拠**：業者卸（法人）と一般エンド（個人）のバランスによる健全性確認
*   **ヒアリング状況**：未
*   **Next Action**：「法人/個人比率が昨年対比で大きく変動しています。市況変化か、特定の営業手法への偏りがないか確認してください」

### 2-2. 現況販売・リフォーム再販 割合
*   **算式**：`COUNT(fact_契約 WHERE 契約種別='売却') GROUP BY dim_出口.現況再販区分 / 全売却契約数`
*   **粒度**：月 / 支店 / 個人
*   **閾値（仮置き）**：昨年同月または全社平均から±15%以上の乖離で警告
*   **根拠**：早期資金回収（現況）と高粗利獲得（リフォーム）の戦略バランスを見るため
*   **ヒアリング状況**：未
*   **Next Action**：「現況/再販比率が通常レンジから乖離しています。粗利率および資金繰りへの影響を確認してください」

### 2-3. リフォーム再販 平均粗利
*   **算式**：`AVG(fact_契約.粗利_確定) WHERE dim_出口.現況再販区分 = 'リフォーム再販'`
*   **粒度**：月 / 支店 / 個人
*   **閾値（仮置き）**：計画値または17期実績の第1四分位未満
*   **根拠**：付加価値を付けた際の絶対的な収益性をモニタリングするため
*   **ヒアリング状況**：未
*   **Next Action**：「リフォーム再販の平均粗利が[閾値]円を下回っています。工事費用の高騰か、販売価格の設定ミスがないか分析してください」

### 2-4. リフォーム再販 平均売却日数
*   **算式**：`AVG(dim_物件.売却決済日 - dim_物件.仕入決済日) WHERE dim_出口.現況再販区分 = 'リフォーム再販'`
*   **粒度**：月 / 支店 / 個人
*   **閾値（仮置き）**：**120日超で警告 / 180日超で要対応**
*   **根拠**：工期を含むため長引きやすいが、資金拘束リスクを管理するため
*   **ヒアリング状況**：未
*   **Next Action**：「リフォーム再販の決済から決済までの平均日数が[閾値]日を超過しています。工事遅延か販売長期化か、ボトルネックを特定してください」

## 【3】契約分析
支店・個人ごとの契約内容の構成、効率、および資本回転の状況を比較分析する。

### 3-1. 買取・仲介比率
*   **算式**：`契約種別（買取・仲介）ごとの件数 / 全契約件数`
*   **粒度**：月 / 支店 / 個人
*   **閾値（仮置き）**：支店ごとの戦略目標値から±10%乖離
*   **根拠**：自社保有リスクを取る買取と、手堅い仲介のバランス指標
*   **ヒアリング状況**：未
*   **Next Action**：「仲介比率が目標より低下しています。買取偏重による資金圧迫リスクがないか確認してください」

### 3-2. サンタメ比率
*   **算式**：`COUNT(WHERE サンタメフラグ=TRUE) / COUNT(fact_契約)`
*   **粒度**：月 / 支店 / 個人
*   **閾値（仮置き）**：17期実績の中央値 ± 1σ を逸脱したら警告
*   **根拠**：第三者のためにする契約（サンタメ）は節税やスピードの観点で重要だが偏りもリスク
*   **ヒアリング状況**：未（※判定ロジック要確認）
*   **Next Action**：「サンタメ比率が通常レンジ[下限-上限]%から逸脱しています。契約スキームの構成を確認してください」

### 3-3. 契約件数 平均値（仕入・仲介・売却）
*   **算式**：`COUNT(fact_契約.契約ID) GROUP BY 契約種別 / 営業月数`
*   **粒度**：月 / 支店 / 個人
*   **閾値（仮置き）**：月次計画値の80%未満
*   **根拠**：各フェーズでの活動量のベースラインを維持するため
*   **ヒアリング状況**：未
*   **Next Action**：「[契約種別]の平均契約件数が計画の[閾値]%を下回っています。パイプラインの停滞を確認してください」

### 3-4. 支店別 1人当たり平均契約数
*   **算式**：`全契約件数 / 営業人員（係数換算後）`
*   **粒度**：月 / 支店
*   **閾値（仮置き）**：全社平均の第1四分位未満
*   **根拠**：支店間の生産性格差を比較・是正するため
*   **ヒアリング状況**：未
*   **Next Action**：「1人当たりの平均契約数が[閾値]件を下回っています。業務効率化や人員スキルの底上げを検討してください」

### 3-5. 1人当たり平均粗利
*   **算式**：`SUM(fact_契約.粗利_確定) / 営業人員（係数換算後）`
*   **粒度**：月 / 支店
*   **閾値（アクション条件）**：**200万円以上で増員アラート** / 第1四分位未満で警告
*   **根拠**：200万/人を超えると既存メンバーの処理キャパシティ上限に達し、機会損失を生むため（指示書より）
*   **ヒアリング状況**：未
*   **Next Action**：「1人当たり平均粗利が200万円を超過しています。支店のキャパシティ上限に達している可能性が高いため、新規採用・増員計画をスケジュールリングしてください」

### 3-6. 在庫回転日数
*   **算式**：`dim_物件.売却日 - dim_物件.取得日` （中央値併記）
*   **粒度**：月 / 支店 / 個人
*   **閾値（仮置き）**：120日超で警告 / 180日超で要対応
*   **根拠**：資本効率の低下と価格下落リスクの管理
*   **ヒアリング状況**：未
*   **Next Action**：「在庫日数[閾値]日超の物件が[N]件あります。価格見直しか出口戦略変更を検討してください」

### 3-7. 在庫回転金額（回転率）
*   **算式**：`期間内の売上原価（売却物件の仕入簿価） / 期間中の平均在庫金額`
*   **粒度**：四半期・年 / 支店
*   **閾値（仮置き）**：全社平均（過去実績）を下回る場合
*   **根拠**：日数だけでなく、金額ベースでの資金効率・投下資本の回転スピードを測るため
*   **ヒアリング状況**：未
*   **Next Action**：「在庫回転金額（率）が悪化しています。高額物件の長期滞留がないか、ポートフォリオを確認してください」

### 3-8. 現金投下粗利率
*   **算式**：`SUM(fact_契約.粗利_確定) / SUM(fact_契約.投下現金額)` （買取契約のみ）
*   **粒度**：月 / 支店 / 個人
*   **閾値（仮置き）**：17期実績の第1四分位を下回ったら要注意
*   **根拠**：手金（現金）をいくら投じていくらリターンを得たかの純粋な資本効率
*   **ヒアリング状況**：未
*   **Next Action**：「現金投下粗利率が[閾値]%を下回っています。仕入価格の妥当性と融資レバレッジの活用状況を確認してください」

## 【4】成長分析
出店計画・採用計画のスケジューリングの判断材料とするため、支店や個人の「成長スピード」と「各種指標（【1】【2】【3】）の移り変わり」を可視化する。

### 4-1. 支店フェーズ別 成長分析
*   **算式**：`dim_支店.開設年月` からの経過月と各種KPI（粗利、契約数など）の推移
    *   **フェーズ定義**：新規出店（1年目） ⇒ 成長支店（2〜3年目） ⇒ 成熟支店（4年目〜）
*   **粒度**：支店 × 経過年・月
*   **閾値（仮置き）**：各フェーズの全社標準成長曲線（過去出店実績）の80%未満
*   **根拠**：新規出店の成功可否の早期判定と、成熟支店のテコ入れ時期の見極め
*   **ヒアリング状況**：未
*   **Next Action**：「自支店は[成長/成熟]フェーズですが、標準的な成長曲線を下回っています。本部と連携し、ボトルネック解消に向けたリソース投入を検討してください」

### 4-2. 入社期別 コホート成長分析
*   **算式**：`入社期（XX期）` ごとの経過月ベースでの累積粗利・1人当たり粗利推移
*   **粒度**：期 × 経過月
*   **閾値（仮置き）**：過去期の標準成長ラインの第1四分位未満
*   **根拠**：会社規模の変遷（期ごとの環境の違い）を考慮しつつ、同期ごとの立ち上がりスピードを比較するため
*   **ヒアリング状況**：未
*   **Next Action**：「[XX期]入社メンバーの成長カーブが標準を下回っています。該当期の研修・OJT体制の見直しを実施してください」

### 4-3. 経験属性別 成長傾向分析
*   **算式**：`dim_社員.前職経験（不動産買取 / 仲介 / 注文住宅 / 未経験 等）` ごとの入社後KPI推移
*   **粒度**：経験属性 × 経過月
*   **閾値（仮置き）**：参考表示（採用戦略へのフィードバック用）
*   **根拠**：どのようなバックグラウンドの社員が自社で早期に立ち上がりやすいか、採用ターゲット選定に活かすため
*   **ヒアリング状況**：未（※社員マスタに前職属性を持たせる必要あり）
*   **Next Action**：「（HR向け）[未経験/特定業種]出身者の立ち上がりデータです。次期採用計画のターゲット要件に反映させてください」

### 4-4. 成長フェーズとKPIの相関推移
*   **算式**：社員の経過月数（成長曲線）を横軸に、上記【1】【2】【3】の代表指標（情報源割合、買取/仲介比率など）の構成比を帯グラフで表示
*   **粒度**：経過年・月
*   **閾値（仮置き）**：特になし（傾向把握用）
*   **根拠**：「初心者は仲介メインから始まり、徐々に高粗利の買取・リフォーム再販へシフトする」といった成長に連動した業務内容の移り変わりを証明し、育成のマイルストーンにするため
*   **ヒアリング状況**：未
*   **Next Action**：「（参考指標）年次ごとの行動変容を可視化しています。次期ステップへ進めていないメンバーへの個別指導に活用してください」

## 全指標サマリ

| カテゴリ | 分析テーマ | 評価軸 | 主な活用目的 |
|---|---|---|---|
| **【1】情報入り口** | 情報源の質・量・推移 | 案件数, 粗利, 成約率, 年次推移 | パイプライン最適化、脱・初期情報源依存 |
| **【2】売却先出口** | 顧客層・販売手法 | 法人/個人比, 現況/リフォーム比, 再販粗利/日数 | 出口戦略の健全性担保、リフォームリスク管理 |
| **【3】契約** | 手法バランス・生産性・資本効率 | 買取/仲介比, 1人粗利(200万基準), 回転日数/金額 | 増員判断、生産性比較、資金効率・滞留アラート |
| **【4】成長** | 支店・個人の立ち上がり速度 | 支店フェーズ, 期別コホート, 前職経験, 指標相関 | 新規出店・採用のスケジューリング、育成指導 |

## 【1】情報入り口分析（追加）

### 1-5. 案件化〜初回契約リードタイム
*   **算式**：`AVG(fact_案件.契約日 - fact_案件.登録日)`（中央値併記）
*   **粒度**：月 / 支店 / 個人 / 情報源
*   **閾値（仮置き）**：情報源別に設定（例：ネット反響は**30日超で警告**）
*   **根拠**：初動対応のスピードを測る。特にプル型情報源（ネット反響等）では時間が長いほど機会損失に直結するため
*   **ヒアリング状況**：未
*   **Next Action**：「情報源[X]における案件化〜初回契約リードタイムが[閾値]日を超えています。初動対応の優先順位と対応フローを見直してください」

### 1-6. 情報源別 コスト調整後ROI
*   **算式**：`SUM(fact_契約.粗利_確定)` / (`COUNT(fact_案件.案件ID)` × `dim_情報源.コスト係数`) （GROUP BY 情報源ID）
*   **粒度**：月 / 支店 / 情報源
*   **閾値（仮置き）**：全情報源の平均ROIを下回る情報源は予算見直し対象
*   **根拠**：単純な粗利額ではなく広告費等の獲得コストを加味した実質チャネル効率を比較し、予算配分を最適化するため
*   **ヒアリング状況**：未（※`dim_情報源.コスト係数` の整備が必要）
*   **Next Action**：「情報源[X]のコスト調整後ROIが全体平均を[Y]%下回っています。獲得コストに対して粗利が不釣り合いなチャネルへの予算配分を見直してください」

## 【2】売却先出口分析（追加）

### 2-5. 出口別 値引き率
*   **算式**：`(想定売価 - fact_契約.売却価格) / 想定売価`（GROUP BY dim_出口.法人個人区分）
*   **粒度**：月 / 支店 / 個人 / 出口区分
*   **閾値（仮置き）**：**5%超で警告**（仮置き；想定売価データがある場合に限る）
*   **根拠**：エンド向けで交渉を続けた末に値下げして業者卸に流れるなど、無駄に利益を削っているケースをあぶり出すため
*   **ヒアリング状況**：未（※想定売価データの整備が前提）
*   **Next Action**：「[個人名]の値引き率が[閾値]%を超えています。最終的に業者卸に流れた経緯と値下げ判断の妥当性を確認してください」

## 【3】契約分析（追加）

### 3-9. 買取契約〜決済リードタイム
*   **算式**：`AVG(fact_契約.決済日 - fact_契約.契約日)` WHERE 契約種別 = '買取'（中央値併記）
*   **粒度**：月 / 支店 / 個人
*   **閾値（仮置き）**：**60日超で警告 / 90日超で要対応**
*   **根拠**：契約後から引き渡し（決済）までの停滞を早期検知し、資金拘束と次工程（リフォーム着工等）の遅延を防ぐため
*   **ヒアリング状況**：未
*   **Next Action**：「買取案件の契約〜決済リードタイムが[閾値]日を超えています。法務・金融手続き等の停滞要因を特定し、早期解消を図ってください」

### 3-10. 種別・エリア別 在庫回転日数と粗利率
*   **算式**：`在庫日数` と `粗利率` を `dim_物件.種別` × `dim_物件.エリア` でクロス集計
*   **粒度**：月 / 支店 / 物件種別 / エリア
*   **閾値（仮置き）**：参考表示（パターン特定用）。在庫日数120日超 かつ 粗利率低下の組み合わせを要対応とする
*   **根拠**：「戸建は粗利が高いが回転が遅い」等、自支店の得意・不得意パターンを特定し、次月の重点仕入ターゲット決定に直結させるため
*   **ヒアリング状況**：未
*   **Next Action**：「[種別/エリア]の在庫回転が長期化しつつ粗利率も低下しています。仕入判断基準を見直し、自支店の得意パターンへの集中を検討してください」

### 3-11. 長期滞留在庫の評価額（金額）比率
*   **算式**：`SUM(在庫評価額 WHERE 在庫日数 > 120) / SUM(全在庫評価額)`（月末スナップショット）
*   **粒度**：月末 / 支店
*   **閾値（仮置き）**：**20%超で警告 / 30%超で要対応**
*   **根拠**：件数ではなく金額（簿価）ベースで資金の塩漬けリスクを可視化し、経営的インパクトの大きい物件の早期損切りを促すため
*   **ヒアリング状況**：未
*   **Next Action**：「120日超の長期滞留在庫が全在庫評価額の[閾値]%を超えています。対象物件の価格改定または損切りを経営判断として早急に実施してください」
